# ladder_viewer 仕様書 (v1)

KV STUDIO のニーモニックリスト（リスト編集 Ctrl+D でコピペできるテキスト）を
markdown コードブロック ```` ```kvlist ```` に貼ると、ラダー図として描画されるツール。

**設計原則: ニーモニックが正、図はクライアント側で人間向けに派生させるだけ。**
Claude との会話ループ（仕様→ニーモニック→図で目視確認→KV STUDIO へ貼り戻し）が
テキストで完全に閉じることが目的。

## 構成（3ファイル＋デモ）

```
ladder_viewer/
  ladder-core.mjs      # ホスト非依存・依存ゼロの ESM（コア）
  ladder-diagram.mjs   # Web Component アダプタ <ladder-diagram>
  demo.html            # ブラウザで開いて即試せるデモ（テキストエリア→即描画）
  test/                # node:test による単体テスト（追加依存なし）
```

- 素の JS (ESM)、ビルド工程なし、依存ゼロ。GitHub に置くだけで配布可能。
- Obsidian 側（知り合い）は同じコアを `registerMarkdownCodeBlockProcessor('kvlist', ...)`
  から呼ぶだけ（アダプタ約20行、あちらの担当）。

## コア API

```js
export function parse(source)              // ニーモニック → ラング列（直並列木）
export function renderSVG(source, opts)    // → SVG 文字列
export function hydrate(containerEl)       // クロスリファレンス等のイベント付与
```

- `opts.comments`: デバイスコメント辞書 `{ "R000": "起動PB", ... }`（KV STUDIO の
  コメントCSV由来を想定）。**インライン `;` コメントが最優先**、辞書はその次。

## 対応命令（実機エクスポート `sample/*.mnm` で検証済み）

構造系のみ正確に実装。それ以外は汎用フォールバック。

| 分類 | 命令 | 挙動 |
|------|------|------|
| ロード | `LD` `LDB` | 新ブロックをスタックへプッシュ（B = b接点） |
| 直列 | `AND` `ANB` | スタック先頭に直列結合 |
| 並列 | `OR` `ORB` | スタック先頭ブロック全体と並列結合 |
| ブロック結合 | `ANL` `ORL` | スタック上位2ブロックを直列/並列マージ（オペランドなし） |
| 分岐 | `MPS` `MRD` `MPP` | 分岐点保存/読み出し/復帰 → fork ノード生成（ネスト可） |
| 出力 | `OUT` `SET` `RES` | コイル確定（SET/RES はコイル内に S/R 表示） |
| 比較 | `LD=` `AND<>.L` 等 | 条件側ボックス。`.L` 等のサフィックス付き可 |
| エッジ | `LDP/LDF` `ANP/ANF` `ORP/ORF` | 接点に ↑/↓ 表示（実機で確認済み） |
| パルス化 | `MEP` `MEF` | 現在の演算結果をパルス化（条件側・オペランドなし、↑/↓記号） |
| 出力連結 | `CON` | 直前の出力ボックスと次の出力を**直列**連結（`LDA→演算→STA` を1行に） |
| ラベル | `LABEL` | 独立した1ラングとして描画。`NCJ` 等のジャンプは出力ボックス |
| その他すべて | `TMR` `TMH` `MOV.L` `@SUB.L` `MDSTRT` … | **未知命令＋オペランドn個 → 出力側ボックス描画**（enumerate しない） |
| 無視 | `END` `ENDH` `NOP` `DEVICE:n` | 描画しない（`DEVICE:` はファイルヘッダ） |

実機エクスポートで判明した仕様（テスト `test/real.test.mjs` に固定済み）:

- **MPS の保存値はブロックスタックに乗る**: `MPS → LD… → ANL → OUT` という
  「分岐内で新ブロックを作り ANL で保存値と結合する」書き方が実在する。
  分岐プレフィックスは fork 側で描画されるため、分岐内スタック1個での ANL は no-op。
- `;MODULE:名前` `;MODULE_TYPE:0` はメタ行（タイトルにしない）。
  `;<h1/>見出し` はタグを剥がしてラングタイトルに。
- コイルデバイスに `@` プレフィックスあり（`OUT @MR100`、微分実行）。
  オペランドの `#100` `+10` `"文字列"`・数値はデバイス扱いしない。

- 条件側/出力側の分類が必要になった命令は、エクスポートする分類表
  （`CONDITION_OPS` セット）に遅延追記していく方式。
- 未知命令・文法エラーでも**壊れず描く**（レニエント）。エラーはラング単位で赤枠表示。

## 記法

```kvlist
; ラング直前の行コメント → ラングのタイトルとして描画
LD    R000   ; 起動PB     ← インラインコメントがデバイスラベルになる
OR    MR000
ANB   R001   ; 停止PB
OUT   MR000  ; 運転保持
```

- 新しい `LD` が来た時点で（出力確定済みなら）ラング区切り。空行は無視。
- ラング区切りの明示記法は設けない（KV STUDIO からの貼り付けをそのまま通すため）。

## 描画

- グリッド配置（セル単位）。左右レール、接点 `-| |-` / b接点 `-|/|-`、コイル `-( )-`。
- デバイス名はシンボル上、コメントはシンボル下（小さめ）。
- 並列はブロック縦積み＋縦連結線。分岐（MPS系）は fork で縦分岐、各枝が右レールへ。
- 出力（コイル/出力ボックス）は右レールに右詰め。条件との間はワイヤで伸縮。
- **色はハードコードしない**: 線・文字は `currentColor`、要素には CSS クラス
  （`.lad-contact` `.lad-coil` `.lad-wire` 等）。ホスト側 CSS でテーマ追従。
  ハイライト色のみ `var(--ladder-hl, 既定色)` でフォールバック付き。

## hydrate（v1 スコープ）

- **クロスリファレンス**: 接点/コイルにホバー → 同一デバイスの要素が
  文書内の全ラング・全 kvlist ブロックで一斉ハイライト（`data-dev` 属性ベース）。
- クリックでコイル定義ラングへジャンプ、リッチツールチップは v2 以降。
  （SVG ネイティブの `<title>` によるコメント表示だけは無料なので入れる）

## KVスクリプト（1行スクリプト/ボックススクリプト）の扱い

実機調査で確定した事実:

- スクリプトはエクスポート時に**代替ラダー（`条件+NCJ #n / 原文コメント＋コンパイル済み命令 / LABEL #n`）
  へ不可逆に置換**される。スクリプト入り .mnm は KV STUDIO に読み戻せない
- ペーストで代替ラダーを入れてもスクリプトボックスには戻らない
- **1回路ブロック(ラング)に置けるスクリプトボックスは1個**（2個目はエラー → パーサが警告を出す）

したがって**スクリプト本文そのものを正**とする:

- **`;=` 記法**: ノート側でスクリプトボックスを書く表現。連続行は同一ボックス
  ```
  LD MR3013
  ;= IF DM100 > 0 THEN
  ;=   DM102 = 0
  ;= ENDIF
  ```
  レンダラーは KV STUDIO と同じ「条件＋原文ボックス」で描画（`data-dev` 付き
  トークン化でスクリプト内デバイスもクロスリファレンス対象）
- **折り畳み**: 既存エクスポートを読む際は `NCJ〜LABEL` の代替ラダーを検出して
  原文コメントのボックス表示に畳む（コンパイル結果は表示しない）。
  `parse(src, { foldScripts: false })` で生表示も可
- **戻し**: KV STUDIO は**リスト編集へのテキスト貼り付け不可**（GUI内コピペは独自
  クリップボード形式）、**.mnm 読み込みは上書き不可**（モジュール削除→読み込みが必要）。
  さらに実機検証で判明した読み込み可能条件:
  - `;SCRIPT_TYPE:` ヘッダ行があると読み込み拒否 → **出力しない**
  - スクリプトの代替ニモーニック（条件+NCJ〜LABEL）があると読み込めない → **除外**し、
    **原文をコメント行 `;本文` として元の位置に残す**（読み込み後にボックスを
    再作成する際の目印。ボックス自体はファイルに乗らないので GUI で手作成）
  
  - スクリプトラングの右側には `SET <空きデバイス>` のダミー出力（出力なしは回路不成立で
    読み込めない。OUT だと複数スクリプトで二重コイル。ビット番号00-15、範囲外は無言で失敗）
  
  `node kvlist2paste.mjs --mnm 出力.mnm ノート.kvlist` がこの形式
  （CP932+CRLF+ヘッダ）を生成する。手順:
  **モジュール削除 → 読み込み → チェックリストに従い SET の隣にスクリプトボックスを追加**
  
  **SET 錨方式（推奨・実機実証済み）**: ダミー SET は削除せず恒久的に残し、スクリプト
  ボックスを並列出力として併設する。この形でエクスポートすると SET＋代替ニモーニックが
  同一ラングに並ぶが、再取り込み時にツールが代替コード（NCJ〜LABEL）と重複原文コメントを
  除去して SET を錨として保持するため、**エクスポート→再生成のサイクルがバイト単位で安定**
  （冪等）。ボックスの再作成は「SET の隣に足すだけ」になる

## v1 の既知の制限

- 構造系命令は実機エクスポート10ファイル（`sample/*.mnm`、エラーゼロ）で検証済み。
  未知の書き方が出たら `ladder-core.mjs` を直して実物をテストに追加する。
- MPS 直後の `OR`（分岐点をまたぐ並列）と、分岐内スタック1個での `ORL` は
  正確に描画できない（エラー表示）。実機サンプルには未出現。
- タイマ等のオペランド付き出力（`TMH T1 #100`）はボックス描画のみ（専用シンボルなし）。
- `.mnm` ファイルは Shift_JIS。コードブロックに貼る時点で UTF-8 になっている前提
  （ファイルを直接読むホストは `TextDecoder('shift_jis')` で変換すること）。
